⚡ DIVIDE & CONQUER
Binary Search

Search a sorted array by repeatedly halving the search space. Much faster than checking every element.

The Big Idea

If the array is sorted, compare your target with the middle element. If the target is smaller, search the left half; if larger, search the right half. Repeat. Each step eliminates half the remaining elements.

Array: [2, 5, 8, 12, 16, 23, 38, 56, 72, 91] Target = 23 Step 1: [2 5 8 12 16 23 38 56 72 91] Low=0 Mid=4 High=9 → arr[4]=16 < 23 → search RIGHT Step 2: [... 38 56 72 91] + [23] Low=5 Mid=7 High=9 → arr[7]=56 > 23 → search LEFT Step 3: Low=5 Mid=5 High=6 → arr[5]=23 ✓ FOUND at index 5
Algorithm (from your lab)
PSEUDOCODE
BinarySearch(arr, low, high, key) { while (low ≤ high) do { mid := (low + high) / 2; if (arr[mid] = key) then return mid; // Found! else if (arr[mid] < key) then low := mid + 1; // Search right half else high := mid - 1; // Search left half } return -1; // Not found }
Time Complexity
Best Case
O(1)
middle element
Average
O(log n)
Worst Case
O(log n)
Java Code
Java
public class BinarySearch {
    static int binarySearch(int[] arr, int key) {
        int low = 0, high = arr.length - 1;
        while (low <= high) {
            int mid = (low + high) / 2;
            if (arr[mid] == key)   return mid;
            else if (arr[mid] < key) low = mid + 1;
            else                   high = mid - 1;
        }
        return -1;
    }

    public static void main(String[] args) {
        int[] arr = {2, 5, 8, 12, 23, 38, 56};
        int result = binarySearch(arr, 23);
        System.out.println(result != -1 ?
            "Found at: " + result : "Not found");
    }
}
⚡ DIVIDE & CONQUER
Merge Sort

Split the array in half, sort each half, then merge them back together. Guaranteed O(n log n).

The Big Idea

Divide: Split array into two halves recursively until single elements. Conquer: Sort each half. Merge: Combine two sorted halves by comparing elements one by one.

Original: [38 27 43 3 9 82 10] Split: [38 27 43] [3 9 82 10] [38] [27 43] [3 9] [82 10] [38][27][43] [3][9] [82][10] Merge: [27 38] → [27 38 43] [3 9] → [3 9 10 82] Final: [3 9 10 27 38 43 82] ✓
Algorithm
MergeSort(arr, low, high) { if (low < high) then { mid := (low + high) / 2; MergeSort(arr, low, mid); // Sort left half MergeSort(arr, mid+1, high); // Sort right half Merge(arr, low, mid, high); // Merge both halves } } Merge(arr, low, mid, high) { // Create temp arrays L[] and R[] i := low; j := mid+1; k := low; while (i ≤ mid and j ≤ high) do { if (L[i] ≤ R[j]) then arr[k++] := L[i++]; else arr[k++] := R[j++]; } // Copy remaining elements }
Time Complexity
Best
O(n log n)
Average
O(n log n)
Worst
O(n log n)

🌟 Always O(n log n) — that's why it's used in production systems. Extra O(n) space needed.

Java Code
Java
static void mergeSort(int[] arr, int l, int r) {
    if (l < r) {
        int m = (l + r) / 2;
        mergeSort(arr, l, m);
        mergeSort(arr, m + 1, r);
        merge(arr, l, m, r);
    }
}

static void merge(int[] arr, int l, int m, int r) {
    int n1=m-l+1, n2=r-m;
    int[] L=new int[n1], R=new int[n2];
    for(int i=0;i<n1;i++) L[i]=arr[l+i];
    for(int j=0;j<n2;j++) R[j]=arr[m+1+j];
    int i=0,j=0,k=l;
    while(i<n1 && j<n2)
        arr[k++]=(L[i]<=R[j])?L[i++]:R[j++];
    while(i<n1) arr[k++]=L[i++];
    while(j<n2) arr[k++]=R[j++];
}
⚡ DIVIDE & CONQUER
Quick Sort

Pick a pivot, put smaller elements left, larger right. Recursively sort each side. In practice, fastest sorting algorithm.

The Big Idea

Choose a pivot element. Rearrange so all elements smaller than pivot come before it, all larger come after. The pivot is now in its final sorted position. Recursively do the same for left and right halves.

Array: [10, 80, 30, 90, 40, 50, 70] Pivot = arr[last] = 70 Partition: i = -1 j=0: arr[0]=10 < 70 → swap arr[0],arr[0] → i=0 → [10 80 30 90 40 50 70] j=2: arr[2]=30 < 70 → swap arr[1],arr[2] → i=1 → [10 30 80 90 40 50 70] j=4: arr[4]=40 < 70 → swap arr[2],arr[4] → i=2 → [10 30 40 90 80 50 70] j=5: arr[5]=50 < 70 → swap arr[3],arr[5] → i=3 → [10 30 40 50 80 90 70] Place pivot: swap arr[4],arr[6] Result: [10 30 40 50 70 90 80] ← pivot 70 in final position!
Algorithm
QuickSort(arr, low, high) { if (low < high) then { pi := Partition(arr, low, high); // pivot index QuickSort(arr, low, pi-1); QuickSort(arr, pi+1, high); } } Partition(arr, low, high) { pivot := arr[high]; i := low - 1; for j := low to high-1 do { if (arr[j] ≤ pivot) then { i := i + 1; swap(arr[i], arr[j]); } } swap(arr[i+1], arr[high]); return i + 1; }
Time Complexity
Best
O(n log n)
Average
O(n log n)
Worst
O(n²)

⚠️ Worst case when array is already sorted. In-place: O(log n) space.

🌐 GRAPH TRAVERSAL
BFS & DFS

Two fundamental ways to visit every node in a graph. BFS explores level by level. DFS goes deep before backtracking.

BFS — Breadth First Search

Uses a Queue. Visit all neighbors of current node before going deeper. Like spreading water — level by level.

Graph: 1→[2,3], 2→[4,5], 3→[6] Queue: [1] Visit 1 → Queue: [2, 3] Visit 2 → Queue: [3, 4, 5] Visit 3 → Queue: [4, 5, 6] Visit 4, 5, 6 → done BFS Order: 1 2 3 4 5 6
DFS — Depth First Search

Uses a Stack (or recursion). Go as deep as possible, then backtrack. Like exploring a maze — one path fully before another.

Same Graph: 1→[2,3], 2→[4,5], 3→[6] Stack: [1] Visit 1 → push 2,3 → Stack: [3, 2] Visit 2 → push 4,5 → Stack: [3, 5, 4] Visit 4 → Stack: [3, 5] Visit 5 → Stack: [3] Visit 3 → push 6 DFS Order: 1 2 4 5 3 6
Algorithm — BFS
BFS(G, start) { for all vertices v: visited[v] := false; Queue Q; Q.enqueue(start); visited[start] := true; while (Q not empty) do { v := Q.dequeue(); print(v); for each neighbor w of v do if (visited[w] = false) then { Q.enqueue(w); visited[w] := true; } } }
Algorithm — DFS
DFS(G, v) { visited[v] := true; print(v); for each neighbor w of v do if (visited[w] = false) then DFS(G, w); // Recursive call }
Java Code
Java
import java.util.*;

public class GraphTraversal {
    static int[][] adj;
    static boolean[] visited;
    static int n;

    static void bfs(int start) {
        Queue<Integer> queue = new LinkedList<>();
        visited[start] = true;
        queue.add(start);
        while (!queue.isEmpty()) {
            int v = queue.poll();
            System.out.print(v + " ");
            for (int i = 1; i <= n; i++)
                if (adj[v][i] == 1 && !visited[i]) {
                    visited[i] = true;
                    queue.add(i);
                }
        }
    }

    static void dfs(int v) {
        visited[v] = true;
        System.out.print(v + " ");
        for (int i = 1; i <= n; i++)
            if (adj[v][i] == 1 && !visited[i])
                dfs(i);
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        n = sc.nextInt();
        adj = new int[n+1][n+1];
        visited = new boolean[n+1];
        int e = sc.nextInt();
        for (int i = 0; i < e; i++) {
            int u = sc.nextInt(), v = sc.nextInt();
            adj[u][v] = adj[v][u] = 1;
        }
        System.out.print("BFS: "); bfs(1);
        Arrays.fill(visited, false);
        System.out.print("\nDFS: "); dfs(1);
    }
}
🌐 GRAPH
Articulation Points

A vertex whose removal increases the number of connected components. Critical nodes — removing them disconnects the graph.

What is an Articulation Point?
💡
Imagine a network of computers. An articulation point is a computer whose failure would split the network into isolated parts. Finding these is critical for network reliability.
Graph: 1-2-3-4-5, with 3 also connected to 6-7 1 --- 2 --- 3 --- 4 --- 5 | 6 | 7 Removing node 2: {1} and {3,4,5,6,7} → DISCONNECTED → Articulation Point! Removing node 3: {1,2} and {4,5} and {6,7} → DISCONNECTED → Articulation Point!
Algorithm — Art(u, v) from your lab
EXACT LAB ALGORITHM
Art(u, v) // u is start vertex for DFS. v is its parent. // dfn[] = DFS discovery number, L[] = low value // num = global counter, initialized to 1 { dfn[u] := num; L[u] := num; num := num + 1; for each vertex w adjacent from u do { if (dfn[w] = 0) then // w is unvisited { Art(w, u); L[u] := min(L[u], L[w]); // u is AP if: u is root with 2+ children // OR L[w] >= dfn[u] for non-root } else if (w ≠ v) then L[u] := min(L[u], dfn[w]); } }
Key Concepts
1
dfn[u] — Discovery time of vertex u in DFS (when it was first visited)
2
L[u] (Low value) — The minimum of dfn[u], dfn of all back-edge endpoints reachable from u's subtree
3
u is Articulation Point if: It's the root of DFS tree with ≥2 children, OR it's non-root and L[child] ≥ dfn[u]
🌐 GRAPH
Biconnected Components

Maximal subgraphs with no articulation points. Each component remains connected even after removing any single vertex.

What is a Biconnected Component?
💡
A biconnected component is a maximal subgraph that has no articulation point. Between any two vertices in a BCC, there are at least two vertex-disjoint paths.
🔗
A graph is biconnected if it is connected and has no articulation points. BCCs are separated at articulation points.
Algorithm (Uses Art + Stack)
// Modified DFS — push edges onto stack BiConnect(u, v) { dfn[u] := num; L[u] := num; num++; for each w adjacent from u do { if (dfn[w] = 0) then { push edge (u,w) onto stack; BiConnect(w, u); L[u] := min(L[u], L[w]); if (L[w] ≥ dfn[u]) then { // u is an articulation point // Pop edges until (u,w) — these form a BCC print "New BCC:"; while top of stack ≠ (u,w): pop & print; pop (u,w); } } else if (w ≠ v and dfn[w] < dfn[u]) then { push edge (u,w); L[u] := min(L[u], dfn[w]); } } }
Key Point

Every edge belongs to exactly one biconnected component. Two BCCs share at most one vertex (an articulation point). The algorithm is essentially DFS + stack of edges.

🗺️ SHORTEST PATH
Dijkstra's Algorithm

Find the shortest path from a source vertex to all other vertices in a weighted graph. Greedy approach with a distance table.

The Big Idea

Maintain a dist[] array. Initially all distances = ∞ except source = 0. Repeatedly pick the unvisited vertex with minimum distance, mark it visited, then update distances of its neighbors.

Graph with source = 1 Vertices: 1,2,3,4,5 Edges: 1-2(10), 1-3(3), 2-4(2), 3-2(4), 3-4(8), 3-5(2), 4-5(5), 5-4(1) Initial: dist = [0, ∞, ∞, ∞, ∞] visited={} Pick 1: Update: dist[2]=10, dist[3]=3 dist = [0, 10, 3, ∞, ∞] Pick 3: (min unvisited) Update: dist[2]=min(10,7)=7, dist[5]=5 dist = [0, 7, 3, ∞, 5] Final: dist = [0, 7, 3, 9, 5] (shortest from vertex 1)
Algorithm (from your lab)
EXACT LAB ALGORITHM
ShortestPaths(v, cost, dist, n) // dist[j] = cost of shortest path from v to j { for i := 1 to n do { S[i] := false; dist[i] := cost[v][i]; } S[v] := true; dist[v] := 0; for num := 2 to n-1 do { // Choose u = unvisited vertex with min dist u := Choose(dist, S, n); S[u] := true; for each w not in S do { if (dist[u] + cost[u][w] < dist[w]) then dist[w] := dist[u] + cost[u][w]; } } }
Time Complexity
Simple Array
O(V²)
Min-Heap
O((V+E)logV)
Fibonacci Heap
O(E + VlogV)

⚠️ Does NOT work with negative edge weights. Use Bellman-Ford for that.

🌲 GREEDY / MST
Prim's Algorithm

Find the Minimum Spanning Tree by greedily adding the cheapest edge that connects a new vertex. Grows one tree from start.

The Big Idea

Start with any vertex. At each step, add the cheapest edge that connects a vertex inside our growing tree to a vertex outside. Stop when all vertices are included.

Graph: 5 vertices, start from vertex 1 Edges: 1-2(2), 1-3(3), 2-3(1), 2-4(4), 3-5(5), 4-5(6) Step 1: Tree={1} Min edge from 1: 1-2(2) → add Step 2: Tree={1,2} Min edge: 2-3(1) → add Step 3: Tree={1,2,3} Min edge: 1-3(3) or 2-4(4)? → 2-4(4) is cheapest new Step 4: Tree={1,2,3,4} Min edge: 3-5(5) → add MST edges: 1-2(2), 2-3(1), 2-4(4), 3-5(5) Total cost = 12
Algorithm (from your lab)
EXACT LAB ALGORITHM
Prim(E, cost, n, t) // t[1..n-1][1..2] stores the MST edges { // Find least cost edge Let (k,l) be edge with minimum cost in E; mincost := cost[k][l]; t[1][1] := k; t[1][2] := l; for i := 1 to n do if (cost[i][l] < cost[i][k]) then near[i] := l; else near[i] := k; near[k] := near[l] := 0; for i := 2 to n-1 do { j := // vertex with min cost[j][near[j]] t[i][1] := j; t[i][2] := near[j]; mincost := mincost + cost[j][near[j]]; near[j] := 0; for k := 1 to n do if (near[k]≠0) and (cost[k][j] < cost[k][near[k]]) then near[k] := j; } return mincost; }
Complexity
Simple
O(V²)
Binary Heap
O(E log V)
Space
O(V+E)
🔧 GREEDY / MST
Kruskal's Algorithm

Build MST by sorting ALL edges and adding cheapest ones that don't form a cycle. Uses Union-Find data structure.

The Big Idea

Sort all edges by weight. Pick the cheapest edge. If adding it doesn't create a cycle, add it to MST. Repeat until n-1 edges selected. Uses Union-Find to detect cycles.

Edges sorted: (1,2,1), (2,3,2), (4,5,2), (1,3,3), (2,4,4), (3,5,5) Step 1: (1,2,1) → no cycle → ADD MST: {1-2} Step 2: (2,3,2) → no cycle → ADD MST: {1-2, 2-3} Step 3: (4,5,2) → no cycle → ADD MST: {1-2, 2-3, 4-5} Step 4: (1,3,3) → CYCLE! (1,2,3 already connected) → SKIP Step 5: (2,4,4) → no cycle → ADD MST: {1-2, 2-3, 4-5, 2-4} MST complete! 4 edges for 5 vertices. Total = 1+2+2+4 = 9
Algorithm (from your lab)
EXACT LAB ALGORITHM
Kruskal(E, cost, n, t) // E = edge set, sorted by cost. n = vertices. { for i := 1 to n do parent[i] := -1; // Init i := 0; mincost := 0; while (i < n-1 and E not empty) do { // Remove cheapest edge from E (u,v) := edge with min cost from E; j := Find(u); k := Find(v); if (j ≠ k) then // No cycle! { i := i + 1; t[i][1] := u; t[i][2] := v; mincost := mincost + cost(u,v); Union(j, k); // Merge components } } if (i ≠ n-1) then print "No spanning tree"; else return mincost; }
Prim vs Kruskal
🌲
Prim's — Grows one connected tree. Better for dense graphs (many edges).
🔧
Kruskal's — Sorts all edges. Better for sparse graphs (few edges).
📋 GREEDY
Job Sequencing

Schedule jobs to maximize total profit, given each job has a deadline and profit. Only one job can run at a time.

The Problem
📌
You have n jobs, each with a deadline and profit. Each job takes 1 unit of time. Maximize total profit by selecting jobs that can finish by their deadlines.
Jobs sorted by profit (decreasing): Job | Deadline | Profit J1 | 2 | 100 ← highest profit J2 | 1 | 50 J3 | 2 | 10 J4 | 1 | 20 Greedy: pick highest profit jobs that fit: Time slots: [_, _] J1 (deadline 2): slot 2 available → assign slot 2 → [_, J1] J2 (deadline 1): slot 1 available → assign slot 1 → [J2, J1] J4 (deadline 1): slot 1 taken, no earlier → SKIP J3 (deadline 2): slot 2 taken → SKIP Selected: J1, J2 Total Profit = 100 + 50 = 150
Algorithm JS(d, j, n) — from your lab
EXACT LAB ALGORITHM
JS(d, j, n) // d[i] = deadlines, p[1]≥p[2]≥...≥p[n] (sorted) // J[i] = ith job in optimal solution { d[0] := J[0] := 0; // Initialize J[1] := 1; k := 1; // Include job 1 for i := 2 to n do { // Find position for job i r := k; while ((d[J[r]] > d[i]) and (d[J[r]] ≠ r)) do r := r - 1; if ((d[J[r]] ≤ d[i]) and (d[i] > r)) then { // Insert job i at position r+1 for q := k to r+1 step -1 do J[q+1] := J[q]; J[r+1] := i; k := k + 1; } } return k; }
Complexity
Time
O(n²)
With Union-Find
O(n log n)
Space
O(n)
🌳 TREES
BST Traversals

Three ways to visit every node in a Binary Search Tree. The order you visit nodes changes what you get.

The Tree
BST with values: 50, 30, 70, 20, 40 50 / \ 30 70 / \ 20 40 Inorder: 20 30 40 50 70 ← sorted order! Preorder: 50 30 20 40 70 ← root first Postorder: 20 40 30 70 50 ← root last
Three Traversals — Simple Memory Trick
IN
Inorder: Left → Root → Right
Gives nodes in sorted ascending order. Used to check if BST is valid. Output: 20 30 40 50 70
PRE
Preorder: Root → Left → Right
Root first. Used to copy/serialize a tree. Output: 50 30 20 40 70
POST
Postorder: Left → Right → Root
Root last. Used to delete a tree (delete children before parent). Output: 20 40 30 70 50
Algorithms (from your lab)
Inorder(root) { if (root ≠ null) then { Inorder(root.left); print(root.data); Inorder(root.right); } }
Preorder(root) { if (root ≠ null) then { print(root.data); Preorder(root.left); Preorder(root.right); } }
Postorder(root) { if (root ≠ null) then { Postorder(root.left); Postorder(root.right); print(root.data); } }
Java Code
Java
class Node {
    int data; Node left, right;
    Node(int d){ data=d; left=right=null; }
}

static Node insert(Node root, int key) {
    if(root==null) return new Node(key);
    if(key < root.data) root.left  = insert(root.left, key);
    else               root.right = insert(root.right, key);
    return root;
}
static void inorder(Node r) {
    if(r!=null){ inorder(r.left); System.out.print(r.data+" "); inorder(r.right); }
}
static void preorder(Node r) {
    if(r!=null){ System.out.print(r.data+" "); preorder(r.left); preorder(r.right); }
}
static void postorder(Node r) {
    if(r!=null){ postorder(r.left); postorder(r.right); System.out.print(r.data+" "); }
}
